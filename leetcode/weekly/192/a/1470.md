## 方法一：创建新数组

创建一个长为 $2n$ 的数组 $\textit{ans}$，作为答案。

根据题意，对于 $i=0,1,2,\ldots,n-1$：

- 把 $\textit{nums}[i]$ 填入 $\textit{ans}[2i]$。
- 把 $\textit{nums}[n+i]$ 填入 $\textit{ans}[2i+1]$。

```py [sol-Python3]
class Solution:
    def shuffle(self, nums: list[int], n: int) -> list[int]:
        ans = [0] * (2 * n)
        for i in range(n):
            ans[i * 2] = nums[i]
            ans[i * 2 + 1] = nums[n + i]
        return ans
```

## 方法二：原地交换

例如 $n=4$：

- $\textit{nums}[0]$ 不变。
- $\textit{nums}[1]$ 要移动到 $\textit{nums}[2]$。
- $\textit{nums}[2]$ 要移动到 $\textit{nums}[4]$。
- $\textit{nums}[4]$ 要移动到 $\textit{nums}[1]$。我们可以按照 $1\to 2\to 4\to 1$ 的顺序走一个环，把环上的每个数移动到其后面的数上。
- $\textit{nums}[3]$ 要移动到 $\textit{nums}[6]$。
- $\textit{nums}[6]$ 要移动到 $\textit{nums}[5]$。
- $\textit{nums}[5]$ 要移动到 $\textit{nums}[3]$。我们可以按照 $3\to 6\to 5\to 3$ 的顺序走一个环，把环上的每个数移动到其后面的数上。
- $\textit{nums}[7]$ 不变。

示例 2 的 $\textit{nums}=[1,2,3,4,4,3,2,1]$，按照上述过程，结果为 $[1,4,2,3,3,2,4,1]$。

如何判断一个数是否已经访问过了？

可以用一个布尔数组 $\textit{vis}$ 标记访问过的下标，但这样需要 $\mathcal{O}(n)$ 的额外空间。

考虑到本题 $\textit{nums}[i]$ 都是正数，我们可以把访问过的数加个负号（变成相反数），这样当我们遍历到一个负数时，可以直接跳过，继续向后遍历。

最后把每个数取反，复原成正数，即为答案。

### 答疑

**问**：这个做法是否会把一个元素标记多次（取反多次），或者有元素没有被标记？

**答**：设 $f(i)$ 是下标为 $i$ 的元素在答案中的下标。根据题意，$f$ 是排列 $[0,1,2,\ldots,2n-1]$ 的一个**置换**。由于置换可以拆分成若干个环，所以每个元素恰好被标记一次。

```py [sol-Python3]
class Solution:
    def shuffle(self, nums: List[int], n: int) -> List[int]:
        for i, x in enumerate(nums):
            if x < 0:  # 已访问
                continue
            cur = i
            while True:
                # 元素 x 要填入 nums[nxt]
                nxt = cur * 2 if cur < n else (cur - n) * 2 + 1
                if nxt == i:  # 回到起点
                    nums[i] = -x  # 用负数表示访问过
                    break
                # 把 x 填入 nums[nxt]（用负数表示访问过）
                # 同时把原来位于 nxt 的数记为 x
                x, nums[nxt] = nums[nxt], -x
                cur = nxt

        # 复原
        for i, x in enumerate(nums):
            nums[i] = -x

        return nums
```

class Solution {
    public int[] shuffle(int[] nums, int n) {

                nums[nxt] = -nums[nxt];
                cur = nxt;
            }

            x = -x;
        }

        return nums;

};

```rust [sol-Rust]
impl Solution {
    pub fn shuffle(mut nums: Vec<i32>, n: i32) -> Vec<i32> {
        let n = n as usize;
        for i in 0..nums.len() {
            let mut x = nums[i];
            if x < 0 { // 已访问
                continue;
            }
            let mut cur = i;
            loop {
                // 元素 x 要填入 nums[nxt]
                let nxt = if cur < n { cur * 2 } else { (cur - n) * 2 + 1 };
                if nxt == i { // 回到起点
                    nums[i] = -x; // 用负数表示访问过
                    break;
                }
                // 把 x 填入 nums[nxt]（用负数表示访问过）
                // 同时把原来位于 nxt 的数记为 x
                (x, nums[nxt]) = (nums[nxt], -x);
                cur = nxt;
            }
        }

        // 复原
        for x in &mut nums {
            *x = -*x;
        }

        nums
    }
}
```

#### 复杂度分析

- 时间复杂度：$\mathcal{O}(n)$。虽然写了个二重循环，但对于每个元素来说，「被标记为负数」只会发生一次（恰好一次），所以总的循环次数是 $\mathcal{O}(n)$ 的。
- 空间复杂度：$\mathcal{O}(1)$。
