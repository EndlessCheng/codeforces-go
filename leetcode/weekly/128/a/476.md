题目让我们把 $\textit{num}$ 取反。

例如二进制 $\textit{num}=11001$，取反后是 $00110$，即十进制的 $6$。

看上去，计算 `~num` 就好了？

但这样做会把更高位的 $0$ 也取反，对于 $32$ 位整数来说，$11001$ 实际是 $00000000000000000000000000011001$，取反后是 $11111111111111111111111111100110$。

所以对于这个例子，要只把 $\textit{num}$ 的低 $5$ 位取反，也就是计算 $\textit{num}$ 和 $11111$ 的异或。

$11111$ 怎么算？设 $w=5$ 是 $\textit{num}$ 的二进制长度，计算 `1 << w` 可以得到 $100000$，再减去 $1$，得到 $11111$。

```py [sol-Python3]
class Solution:
    def findComplement(self, num: int) -> int:
        w = num.bit_length()
        return ((1 << w) - 1) ^ num
```

```java [sol-Java]
class Solution {
    public int findComplement(int num) {
        int w = 32 - Integer.numberOfLeadingZeros(num);
        return ((1 << w) - 1) ^ num;
    }
}
```

```cpp [sol-C++]
class Solution {
public:
    int findComplement(int num) {
        int w = bit_width((uint32_t) num);
        return ((1u << w) - 1) ^ num;
    }
};
```

```c [sol-C]
int findComplement(int num) {
    int w = 32 - __builtin_clz(num);
    return ((1u << w) - 1) ^ num;
}
```

```go [sol-Go]
func findComplement(num int) int {
	w := bits.Len(uint(num))
	return 1<<w - 1 ^ num
}
```

```js [sol-JavaScript]
var findComplement = function(num) {
    const w = 32 - Math.clz32(num);
    return ((1 << w) - 1) ^ num;
};
```

```rust [sol-Rust]
impl Solution {
    pub fn find_complement(num: i32) -> i32 {
        let w = num.ilog2() + 1;
        ((1 << w) - 1) ^ num
    }
}
```

#### 复杂度分析

- 时间复杂度：$\mathcal{O}(1)$。
- 空间复杂度：$\mathcal{O}(1)$。
