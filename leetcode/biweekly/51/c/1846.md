本题的贪心策略是简单的，难点在证明。

**贪心策略**：把 $\textit{arr}$ 从小到大排序。把 $\textit{arr}[0]$ 变成 $1$。向右遍历后续元素，如果 $\textit{arr}[i] > \textit{arr}[i-1]+1$，则把 $\textit{arr}[i]$ 减小为 $\textit{arr}[i-1]+1$，否则不变。答案为 $\textit{arr}[n-1]$。

证明如下。

首先证明，把一个合法序列（相邻元素绝对差 $\le 1$）升序排序后，仍然是合法序列。

用反证法证明。设升序排序后的序列为 $b$。如果存在 $b[i]-b[i-1] \ge 2$，由于 $b$ 是递增的，所以序列不包含 $b[i-1]+1$。但在排序前的序列中，由于相邻元素绝对差 $\le 1$，从 $b[i-1]$ 到 $b[i]$ 必然会经过 $b[i-1]+1$（联想一下介值定理），说明序列一定包含 $b[i-1]+1$，矛盾，故原命题成立。

因此，任何合法的最终序列，将其排序后仍然合法。所以我们只需讨论升序的最终序列。

设 $a$ 是 $\textit{arr}$ 升序排序后的结果，$b$ 是最终序列升序排序后的结果。

因为 $b$ 是 $a$ 通过减小操作和重排操作得到的，所以必然存在一个一一对应关系。具体地，存在一个 $0$ 到 $n-1$ 的排列 $p$，使得 $b[i]$ 是由 $a[p[i]]$ 减小得到的，即

$$
b[i]\le a[p[i]]
$$

我们要证明：排列 $p = [0,1,2,\ldots,n-1]$ 一定能涵盖最优解。换句话说，存在最优解，满足 $b[i]\le a[i]$ 对每个 $i$ 都成立。

用**交换论证法**证明。

假设排列 $q$ 可以得到最优解，但其中存在下标 $i$ 和 $j$，满足 $i < j$ 且 $q[i] > q[j]$。

由于 $a$ 和 $b$ 都是升序，下标越大，元素值越大（或不变）。所以有

$$
\begin{aligned}
& b[i]\le b[j]      \\
& a[q[j]] \le a[q[i]]    \\
\end{aligned}
$$

此外，由于 $b[i]$ 是 $a[q[i]]$ 减小得到的，$b[j]$ 是 $a[q[j]]$ 减小得到的，所以有

$$
\begin{aligned}
& b[i] \le a[q[i]]    \\
& b[j] \le a[q[j]]    \\
\end{aligned}
$$

能否交换对应关系？让 $b[i]$ 是从 $a[q[j]]$ 减小得到的，$b[j]$ 是从 $a[q[i]]$ 减小得到的。

联立上述不等式，得

$$
\begin{aligned}
& b[i] \le b[j] \le a[q[j]]    \\
& b[j] \le a[q[j]] \le a[q[i]]    \\
\end{aligned}
$$

所以可以交换。

注意交换并没有改变 $b$ 中的元素值，所以 $\max(b)$ 是不变的。

于是，通过有限次交换，我们总是可以把一个合法的排列 $q$ 变成一个升序排列 $[0,1,2,\ldots,n-1]$，同时保持最优解不变。

综上所述，把 $\textit{arr}$ 升序排序后，只需考虑如何减小 $\textit{arr}[i]$ 以满足要求，无需再重新排列 $\textit{arr}$。由于题目要求 $\textit{arr}[0]=1$ 且后续元素绝对差 $\le 1$，所以 $\textit{arr}[1]$ 不能超过 $\textit{arr}[0]+1 = 2$。如果 $\textit{arr}[1] > 2$，那么把 $\textit{arr}[1]$ 减小为 $2$，否则不变。后续元素依此类推，即对于 $1\le i\le n-1$，有递推式

$$
\textit{arr}[i] = \min(\textit{arr}[i], \textit{arr}[i-1] + 1)
$$

最终答案为 $\textit{arr}[n-1]$。

## 写法一

```py [sol-Python3]
class Solution:
    def maximumElementAfterDecrementingAndRearranging(self, arr: List[int]) -> int:
        arr.sort()
        arr[0] = 1
        for i in range(1, len(arr)):
            arr[i] = min(arr[i], arr[i - 1] + 1)
        return arr[-1]
```

#### 复杂度分析

- 时间复杂度：$\mathcal{O}(n\log n)$，其中 $n$ 是 $\textit{arr}$ 的长度。瓶颈在排序上。
- 空间复杂度：$\mathcal{O}(1)$。忽略排序的栈开销。

## 写法二

对于最终序列，从左到右看，从 $1$ 开始，每次至多增加 $1$，所以答案至多为 $n$。

这意味着，原数组 $\textit{arr}$ 中的大于 $n$ 的数都要减小到 $\le n$。所以 $\ge n$ 的数可以视作 $n$。这样我们就可以用 $\mathcal{O}(n)$ 的**计数排序**了。

设 $\textit{cnt}[x]$ 为 $\textit{arr}$ 中的 $x$ 的个数。

和写法一一样，从小到大遍历。例如现在遍历完了所有 $\le 4$ 的数，得到的最大值为 $2$。继续遍历：

- 如果 $\textit{cnt}[5] = 2$，新的最大值是多少？第一个 $5$ 减小为 $3$，第二个 $5$ 减小为 $4$，所以新的最大值为 $4$。
- 如果 $\textit{cnt}[5] = 4$，新的最大值是多少？第一个 $5$ 减小为 $3$，第二个 $5$ 减小为 $4$，第三个 $5$ 不变，第四个 $5$ 不变，所以新的最大值为 $5$。

一般地，设遍历到 $\textit{cnt}[x]$ 之前，最大值为 $\textit{ans}$。利用 $x$，我们可以把 $\textit{ans}$ 增大至 $\textit{ans}+\textit{cnt}[x]$，但这不能超过 $x$，所以有

$$
\textit{ans} = \min(\textit{ans}+\textit{cnt}[x], x)
$$

```py [sol-Python3]
class Solution:
    def maximumElementAfterDecrementingAndRearranging(self, arr: List[int]) -> int:
        n = len(arr)
        cnt = [0] * (n + 1)
        for x in arr:
            cnt[min(x, n)] += 1

        ans = 0
        for x in range(1, n + 1):
            ans = min(ans + cnt[x], x)
        return ans
```
